# Contributing

Thanks for your interest in improving an AKA Security project.

- **Issues** — use the issue templates. Include repro steps for bugs and a clear
  use case for feature requests.
- **Pull requests** — branch from `main`, keep changes focused, and follow the
  commit convention (`feat:`/`fix:`/`docs:`/`chore:` with a lowercase imperative
  summary). CI and review gates must pass before merge.
- **Security** — do not open a public issue for vulnerabilities. See
  [SECURITY.md](SECURITY.md).

Individual repositories may add their own `CONTRIBUTING.md` with
project-specific build and test instructions, which takes precedence over this
one.
